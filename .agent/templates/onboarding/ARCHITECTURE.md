## 5. Architecture Principles

1. **Separation of Concerns** — Each layer has a single responsibility
2. **Dependency Direction** — Dependencies point inward (UI → Business → Data)
3. **Interface Segregation** — Depend on abstractions, not implementations
4. **Immutability** — Prefer immutable data structures
5. **Fail Fast** — Validate early, error explicitly
